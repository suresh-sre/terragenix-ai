# Antigravity Terraform Generator (Python Edition)

A powerful AI-driven tool to generate production-ready Terraform infrastructure code for AWS, Azure, and GCP using Google Gemini models.

## 🚀 Features

- **Multi-Cloud Support**: Generate infrastructure for AWS, Azure, and Google Cloud Platform (GCP).
- **AI-Powered**: Uses advanced Google Gemini models (`gemini-2.0-flash`, `gemini-1.5-pro`, etc.) for intelligent code generation.
- **Dual Interface**:
  - **CLI Tool**: For automation, scripts, and quick generation.
  - **Web Application**: A beautiful, interactive web UI for easy visual generation.
- **Complete Output**: Generates `provider.tf`, `variables.tf`, `main.tf`, `outputs.tf`, `terraform.tfvars.example`, and `README.md`.
- **Flexible Export**: Save as individual files, a single bundle text file, or download as a ZIP archive (Web).

## 📦 Installation

1. **Prerequisites**: Python 3.8+ installed.
2. **Install Dependencies**:
   ```bash
   pip install -r requirements.txt
   ```
   *(Requires `google-generativeai` and `flask`)*

## 🔑 API Key Setup

You need a Google Gemini API key. Get one from [Google AI Studio](https://aistudio.google.com/app/apikey).

- **Option 1 (Env Var)**: `export GEMINI_API_KEY="your_key_here"` (Linux/Mac) or `$env:GEMINI_API_KEY="your_key_here"` (Windows PowerShell)
- **Option 2 (CLI Flag)**: Pass via `-k "your_key_here"` argument.
- **Option 3 (Web UI)**: Enter directly in the web interface.

## 💻 Usage

### 1. Web Application (Recommended)

Start the web server for an interactive experience:

```bash
python antigravity_terraform_generator_web.py
```

- Open **http://localhost:5000** in your browser.
- Enter your API Key, select Cloud Provider, enter Services (e.g., `ec2, rds`), and click **Generate**.
- **Download ZIP** to get all files ready to use.

### 2. Command Line Interface (CLI)

Run the generator directly from your terminal.

**Syntax:**
```bash
python antigravity_terraform_generator.py -k "YOUR_KEY" -p [aws|azure|gcp] -s "service1,service2" [options]
```

**Examples:**

- **Generate AWS Infrastructure (Save to Directory):**
  ```bash
  python antigravity_terraform_generator.py -k "AIzaSy..." -p aws -s "ec2,rds,vpc" -o ./my-aws-infra
  ```
  *Creates a folder `my-aws-infra` with all Terraform files.*

- **Generate Azure Infrastructure (Save as Bundle File):**
  ```bash
  python antigravity_terraform_generator.py -k "AIzaSy..." -p azure -s "vm,sql" -b azure-bundle.txt
  ```

- **Generate GCP Infrastructure (Print to Console):**
  ```bash
  python antigravity_terraform_generator.py -k "AIzaSy..." -p gcp -s "compute,storage"
  ```

## 📂 Output Structure

When saving to a directory (CLI) or downloading ZIP (Web), you get:

- `provider.tf`: Cloud provider configuration.
- `variables.tf`: Input variables definition.
- `main.tf`: The core infrastructure resources.
- `outputs.tf`: Output values (IPs, IDs, etc.).
- `terraform.tfvars.example`: Example variable values.
- `README.md`: Documentation for the generated code.

## 🛠️ Troubleshooting

- **"Model not found"**: The tool automatically tries multiple Gemini models (`gemini-2.0-flash`, `gemini-1.5-flash`, etc.). Ensure your API key has access to these models.
- **"500 Internal Server Error"**: Check your terminal for detailed error logs. Usually indicates an invalid API key or network issue.
